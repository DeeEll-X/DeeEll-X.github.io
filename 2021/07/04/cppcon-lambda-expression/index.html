  1. lambda: fundamental definition in c++: an expression which returns a function object

  2. 函数指针：

    • pointer to any funciton
    • 不能被解引用，常被用于回调函数或另一个函数的参数

  3. 函数对象（function object）

    • function object data type:指拥有operator()（a function call operator）方法的一个类或结构体。
    • function object:指一个function object data type的一个实例。调用方法和普通函数一样。

  4. lambda expression：

    • 定义了一个funciton object
    • 常被赋值给一个auto类型的变量，auto的自动类型推断发生在编译期，所以使用auto不会造成程序运行时效率降低
    • closure(闭包)是计算lambda表达式返回的一个function object。closure的数据类型就是lambda表达式被auto推断的数据类型。
    • key elements:[capture clause](parameter list) -> return type {body}
      • return type一般是自动推断的。
      • capture clause：
        • 值传递：捕获const值，捕获参数被拷贝进function object，之后该参数是否消亡并不影响。注意在body里面修改了捕获值的话：
          auto myLamb = [x]()mutable{return ++x;}
        • 引用传递：参数前加&，只能捕获左值，右值invalid。捕获的左值引用在closure的整个生命周期都应该活着，否则会导致undefined behavior。
          auto myLamb = [&x](){return ++x;}
        • c++14中引入了generalized capture，其中可以使用右值[varA = 10], [&varC = y],[varD = std::move(z)]
        • [=]capture lambda表达式中所有的变量by value
        • [&]capture lambda表达式中所有的变量by reference
        • 全局变量不能被捕获，也不需要，可以直接用
      • parameter list：
        • 数据类型可以是auto或者是模板参数T（generic lambda）
      • return type deduction：如果有多于一条return语句，返回值类型忽略的话返回值要能够被推断成一种类型，要么就明确声明
